## 1. 背景与问题

很多人有打鼾、磨牙、说梦话等睡眠中发声问题，但缺乏低成本、可量化、可追踪的工具：

- 医院睡眠监测（PSG）昂贵、需预约、不便长期跟踪；
- 现有手机 App 多为黑盒打分，**不透明、难以复核、无法二次开发**；
- 用户已经用 Plaud 等设备录了整夜音频，却缺少把"录音"转化为"可读睡眠洞察"的工具。

**核心问题**：如何把一段（或连续多段）整夜录音，自动转化为「每晚睡眠状态 + 多晚趋势（变好/变差）」的结构化洞察，并能被任意 AI 助手直接调用。

---

## 2. 目标与非目标

### 2.1 目标
- 输入一段整夜录音，自动识别**打鼾 / 磨牙 / 梦话**三类事件，输出统计、带时间戳的事件、健康建议。
- 给出**每晚睡眠分（0~100）**与等级，使不同夜晚可比较。
- 支持**连续多份录音**的批量分析，并对多晚做**趋势对比**，判定整体**变好 / 变差 / 持平**及主因。
- 以 **MCP 工具**形式暴露能力，任意支持 MCP 的 AI 助手（Claude Desktop / Claude Code 等）可直接调用，无需写代码。
- 提供 Web 界面用于上传与可视化查看。

### 2.2 非目标（v1 不做）
- 不做医疗级诊断、不做呼吸暂停（OSA）医学判定。
- 不做云端托管 / 多用户账号体系（v1 为本地运行）。
- 不做实时流式分析（v1 为离线整段分析）。

## 3. 目标用户与典型场景

| 用户 | 场景 |
|---|---|
| 关注睡眠质量的个人 | 想知道自己打鼾/磨牙严重程度，以及最近几晚是变好还是变差 |
| 用 Plaud 等设备录整夜音频的用户 | 已有录音，想要可量化的睡眠洞察 |
| 开发者 / 极客 | 想要一个透明、可改、可接入 AI 助手的开源方案 |

**主线场景**：用户连续几晚录音 → 批量分析 → 得到每晚睡眠分 → 一句话问 AI"我这几晚睡眠变好还是变差" → 得到趋势结论与主因。

## 4. 核心功能

### 4.1 声音识别引擎（三类事件）
识别并统计三类睡眠发声：**打鼾(snoring)、磨牙(grinding)、梦话(talking)**，输出每类的次数、时长、占比，以及带**真实录音时间戳**的事件列表。

**三种分类后端**（可切换）：

| 后端 | 方式 | 特点 |
|---|---|---|
| `hybrid`（默认） | YAMNet 测打鼾/梦话 + 规则测磨牙 | 准确率最高，兼顾两者所长 |
| `yamnet` | Google AudioSet 预训练 YAMNet | 打鼾/梦话强；磨牙基本测不到（AudioSet 无对应类） |
| `rule` | 手工特征 + 阈值规则 | 零额外依赖，开箱即用；准确率有限 |

> 依赖缺失（无 tensorflow）时自动回退到 `rule` 并告警，结果 `metadata.backend` 标记实际生效后端。

### 4.2 每晚睡眠分
把各类声音按"每小时次数"归一化后加权，得到 **0~100 睡眠分**（越高越好）与等级（优/良/一般/差）。权重集中在 `config.SCORE_WEIGHTS`，**透明可调**。

### 4.3 多晚趋势分析
多份报告按**录音日期**排序后比较，输出：
- `overall_direction`：`improving` / `worsening` / `stable`（变好/变差/持平）；
- 各指标（睡眠分、打鼾/h、磨牙/h、梦话/h、发声占比）的首末变化与方向；
- 一段可直接展示的中文结论。
- 睡眠分变化超过 `TREND_SCORE_DELTA`（默认 5 分）才判定为变好/变差，否则视为持平。

### 4.4 报告存档与秒级复用
每次分析可保存为 JSON 报告；趋势分析直接读取历史报告，**无需重跑音频**（实测秒级）。

### 4.5 MCP 工具清单
| 工具 | 作用 |
|---|---|
| `analyze_sleep_audio(audio_path, …, backend="hybrid")` | 分析单晚，返回统计/事件/建议/睡眠分 |
| `analyze_sleep_batch(audio_paths, …)` | 连续多份分析 + 多晚趋势 |
| `analyze_sleep_trend(filenames/date_from/date_to, only_dated=True)` | 对历史报告做趋势，免重跑 |
| `list_sleep_reports()` / `get_sleep_report(filename)` | 列出/读取历史报告 |

### 4.6 Web 界面
本地 Flask 应用：上传录音、查看统计卡片、事件列表、健康建议、历史报告。

## 5. 技术架构

```
录音(wav/mp3/m4a)
  → 预处理(16kHz 重采样 / 降噪 / 归一化 / 去静音 / 分帧)
  → 分类后端(hybrid: YAMNet 推理 + 规则补磨牙)
  → 事件合并(时间戳映射回真实录音时间)
  → 统计 + 健康建议 + 每晚评分
  → [可选] 多晚趋势分析
  → 输出 JSON 报告 / MCP 返回 / Web 展示
```

- 语言/依赖：Python ≥3.11；librosa、numpy/scipy、tensorflow + tensorflow-hub（YAMNet，可选）、mcp、flask。
- 传输：MCP stdio；Web 为本地 HTTP（默认 5050）。
- 性能：约 1 小时录音 ≈ 10 秒分析（CPU，远快于实时）；趋势读盘 < 1 秒。

## 6. 关键指标与验收

| 维度 | 目标/现状 |
|---|---|
| 分析速度 | 3.86h 录音约 40s（CPU，Apple Silicon），远快于实时 |
| 趋势复用 | 历史报告读盘出趋势 < 1s（实测 0.94s） |
| 打鼾召回 | hybrid 用 AudioSet `Snoring` 类，显著优于规则（规则曾整夜漏检 0 次 → hybrid 117 次） |
| 噪声鲁棒 | hybrid 能区分环境噪声与梦话（规则曾把噪声误判为 347s 梦话 → hybrid 22.5s） |
| 准确率量化 | ⚠️ 待办：目前无带标注测试集，未给出 precision/recall |

## 7. 案例展示（真实数据）

### 7.1 单晚分析（2026-06-02，3.86 小时）
| 指标 | 结果 |
|---|---|
| 睡眠分 | **66.7（一般）** |
| 打鼾 | 117 次 / 73.5s（30.3 次/h） |
| 磨牙 | 15 次 / 10.2s |
| 梦话 | 34 次 / 17.5s |
| 发声占比 | 0.73% |

**后端对比（同一段录音）**：规则后端整夜漏检打鼾（0 次）、把噪声误判为梦话（347.5s）；YAMNet/hybrid 抓到 117 次打鼾、梦话收敛到 ~18s —— 说明预训练模型显著优于规则。

### 7.2 多晚趋势（2026-06-02 → 06-03）
| 日期 | 睡眠分 | 等级 | 时长 | 打鼾/h | 磨牙/h | 梦话/h | 发声占比 |
|---|---|---|---|---|---|---|---|
| 06-02 | 66.7 | 一般 | 3.86h | 30.3 | 3.9 | 8.8 | 0.73% |
| 06-03 | **41.1** | **差** | 4.17h | **58.5** | 6.5 | 8.6 | 1.28% |

**结论**：睡眠分 66.7 → 41.1（**−25.6**），整体 **变差**。
**主因**：打鼾几乎翻倍（30→59 次/h），磨牙也增多；梦话基本持平。两晚时长相近，排除了时长因素。

> 这正是产品价值：**自动判定"变差"并定位到主因（打鼾激增）**，给出可执行方向（如保持侧卧、睡前避免饮酒）。

## 8. 已知局限与风险

1. **磨牙是弱项**：AudioSet 无干净的"夜磨牙(bruxism)"类，hybrid 用规则兜底，召回偏低。
2. **超短录音评分不稳**：1.5 分钟级别的片段"每小时次数"会失真（待加最短时长保护）。
3. **去静音阈值偏激进**：可能漏掉很轻的声音（`top_db` 可调）。
4. **音频是本地路径**：MCP/Desktop 端只能传本机文件路径，无法上传文件给本地 server。
5. **依赖较重**：tensorflow/librosa 在部分机器安装困难（计划提供 Docker 版）。
6. **无量化准确率**：缺带标注测试集，准确率尚未用 precision/recall 量化。

## 10. 附录

### 10.1 安装（开发者）
```bash
git clone https://github.com/Hyperianfan/sleep-sound-analyzer.git
cd sleep-sound-analyzer
python3 -m venv venv && source venv/bin/activate
pip install ".[yamnet]"     # 高准确率后端；仅 rule 用 pip install .
```

### 10.2 在 Claude Desktop 安装
编辑 `claude_desktop_config.json`（macOS：`~/Library/Application Support/Claude/`），加入：
```jsonc
{
  "mcpServers": {
    "sleep-sound-analyzer": {
      "command": "/ABS/PATH/sleep-sound-analyzer/venv/bin/python",
      "args": ["/ABS/PATH/sleep-sound-analyzer/mcp_server.py"]
    }
  }
}
```
重启后即可对 AI 说"帮我分析这段睡眠录音 /path/xxx.wav"。

### 10.3 典型用法（对 AI 助手说）
- "分析这段录音 /path/a.wav"
- "连续分析这几晚 /path/a.wav /path/b.wav，看睡眠是变好还是变差"
- "对比我 2026-06-01 到 06-30 保存的报告，出睡眠趋势"
